# Getting Started

The fastest way to understand VELA is through the official starter kit. It includes all working components and step-by-step instructions for running a complete local environment.

## Starter Kit

The [vela-starterkit](https://github.com/HorizenOfficial/vela-starterkit) repository contains everything you need to spin up a local VELA environment, deploy a sample WASM application, and interact with the system end to end.

Start here if you are new to VELA.

## Public Repositories

All core VELA repositories are public and available for exploration, usage, and feedback.

| Repository | Description |
|---|---|
| [vela](https://github.com/HorizenOfficial/vela) | Core protocol |
| [vela-common-ts](https://github.com/HorizenOfficial/vela-common-ts) | TypeScript shared library |
| [vela-common-go](https://github.com/HorizenOfficial/vela-common-go) | Go shared library |
| [vela-nova](https://github.com/HorizenOfficial/vela-nova) | Nova component |

## What to Expect

VELA is currently in a local-only development phase. You will be working with:

- A Docker-based local environment
- An emulated TEE (not hardware enclaves)
- A single WASM application per deployment

For a full list of current constraints, see [What's Not Supported Yet](/vela/limitations/limitations).

## Feedback

The team is actively building and collecting developer feedback. If you run into issues or have feature requests, open an issue on the [VELA repository](https://github.com/HorizenOfficial/vela) or reach out on [Discord](https://discord.gg/horizen).
