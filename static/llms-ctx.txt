# Horizen — Quick Reference for AI Coding Assistants

> Horizen is an EVM-identical L3 on Base (chain ID 26514) using the OP Stack. Same Solidity, same tooling (Foundry/Hardhat/ethers.js/wagmi). It adds VELA — a confidential execution coprocessor using TEEs for privacy-preserving, compliance-auditable apps.
>
> Use this file for fast answers. For complete docs, see /llms-full.txt

## Deploy a Contract (EVM — no changes needed from Base)

- [Deploy with Foundry](https://docs.horizen.io/horizen-chain/deploy-contracts/using-foundry): Install forge, write contract, deploy with `forge create --rpc-url`, verify
- [Deploy with Hardhat](https://docs.horizen.io/horizen-chain/deploy-contracts/using-hardhat): Configure hardhat.config.ts with Horizen network, deploy with ignition/scripts
- [Verify a Contract](https://docs.horizen.io/horizen-chain/deploy-contracts/verify-contract): Source verification on the Horizen block explorer

## Tokens & Gas

- [Gas on Horizen](https://docs.horizen.io/horizen-chain/tokens-and-gas/gas-on-horizen): ETH for gas (L2+L1 fee model), typical costs
- [ZEN Token](https://docs.horizen.io/horizen-chain/tokens-and-gas/zen-token): ERC-20 governance token on Base (0xf43eB8De897Fbc7F2502483B2Bef7Bb9EA179229)

- [cBTC on Horizen](https://docs.horizen.io/horizen-chain/tokens-and-gas/cbtc): Coinbase-wrapped Bitcoin (cbBTC) bridged to Horizen

## Bridging

- [How Bridging Works](https://docs.horizen.io/horizen-chain/bridging/how-bridging-works): Two bridges available — native OP Stack bridge (ETH only, 7-day withdrawal window) and Stargate (ZEN/USDC/cbBTC, 80+ chains, ~2 min)
- [Bridge Assets via Native Bridge](https://docs.horizen.io/horizen-chain/bridging/bridge-assets): Step-by-step guide for depositing ETH from Base to Horizen and withdrawing back — uses the OP Stack canonical bridge
- [Bridge Assets via Stargate](https://docs.horizen.io/horizen-chain/bridging/bridge-assets-stargate): Cross-chain bridge for ZEN, USDC, cbBTC via LayerZero OFT standard — burn/mint across 80+ chains, ~2 min transfers, 0.06% fee, no wrapping

## VELA (Confidential Execution — TEE coprocessor)

- [What is VELA?](https://docs.horizen.io/vela/overview/what-is-vela): Confidential compute via TEEs, cryptographic attestation, compliance without data exposure
- [Prerequisites](https://docs.horizen.io/vela/getting-started/prerequisites): Required tools and setup before starting VELA development
- [Local Environment Setup](https://docs.horizen.io/vela/getting-started/local-environment-setup): Docker-based local dev environment (VELA not yet on testnet/mainnet)
- [Your First Confidential App](https://docs.horizen.io/vela/getting-started/hello-world): Deploy a private transfer app, deposit ETH into encrypted TEE account, verify private balance
- [Current Limitations](https://docs.horizen.io/vela/limitations/limitations): VELA is local/Docker only; no testnet or mainnet deployment yet; emulated TEE in dev
